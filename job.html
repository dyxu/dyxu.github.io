大众点评网架构组 （ 2014年7月 ~ 现在）

移动端长连接项目服务端

该项目服务端需要连接全国点评 Android 用户和 iOS 用户，为他们提供推送服务和 HTTP 代理服务。项目最大的难度在于需要同时关注总连接数、QPS 和延时，同时也要做各种 JVM 优化，来支持该程序。
该项目由主要我一个人负责。我在期间学习了 Java 网络编程、Netty、Java 并发编程、JVM 性能调优等相关技术。
压测结果：程序最大连接数可到 60 万，单机推送 2 万每秒，单机 HTTP 代理请求 1 万每秒。
最终该项目成功上线，并已平稳运行半年。线上 20 台机器，每天发送2000万推送，为整个点评的移动客户端提供HTTP代理服务，将移动端 API 可用性从 95% 提升至 99%。并获得了公司内部相关奖项。

数据库可用性相关项目

该项目包含一系列子项目，包括读写分离、分库分表、数据库集中配置中心、数据库同步、MHA 切换、数据库监控等。最终目的是提升数据库可用性。
我在项目中主要和另外以为同事搭档，负责所有 Java 相关的开发工作。我们两人分工开发，相互 Review 代码。我在期间学习了 JDBC 规范、读写分离、分库分表、Mysql 复制等相关技术。
目前，分库分表项目和数据库同步项目还在开发中。其余项目均已开发完成并在线上推广。
通过各种项目的配合，我们最终实现了2015年上半年数据库无 S1、S2 故障的目标。并获得了公司内部的相关奖项。

大众点评网CRM团队 （ 2011年8月 ~ 2014年6月 ）

CRM

公司在 2011 年开始采购 Salesforce 相关服务，并开始开发新的 CRM 系统。Salesforce 有自己的语言、程序和数据库，所以如果要和内部打通的话就需要做很多二次开发和数据同步的工作。
我在该项目中主要负责 Salesforce 系统和点评内部系统对接和数据同步。并在其中学到了各种 RPC 和 ETL 相关技术。
该项目迭代开发了多年，一次次地提升着点评销售的工作效率。并与2012年年会上获得了公司最高团队奖项。

销售手机客户端项目

在移动互联网的大趋势下，销售也需要通过各种手机客户端提升工作效率。而销售手机客户端项目最大的难点在于我们团队人数不多，且并没有配备移动端开发工程师。
部门领导派我带领着一支临时组建的小团队一起来调研 PhoneGap 混合开发的可性能。我们在一个月内开发出了第一个 Demo。然后我又负责继续研究 PhoneGap，并和团队一起用 PhoneGap 开发出了点评内部第一个销售手机客户端。
除此以外，为了实现 Android 和 iOS 双平台推送系统，我学习了 iOS 和 Android 开发，做出了两个平台的推送插件。同时也设计和实现了一套同时兼容两个平台的推送系统服务端。
另外 PhoneGap 本身没有统计功能，我也设计并开发了两个平台 Google Analytics PhoneGap 插件，并已开源。
该项目上线后广受好评，特别是推送系统，能让销售更快捷地知道各种审批结果，也能让销售主管更快更方便地在手机上进行审批。大大提升了销售们的工作效率。
